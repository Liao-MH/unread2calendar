# Repository Instructions

## Completion Requirements

For every completed change in this repository, Codex must do all of the following before claiming the work is finished:

1. Confirm that all of the user's requirements are complete, using `✅` checklist-style wording in the final response.
2. Update `docs/CHANGELOG.md` to record the completed change.
3. Produce a new `.xpi` build artifact for the updated Thunderbird add-on.
4. Create a Git commit for the completed development change.

These steps are mandatory for every completed implementation task in this repository unless the user explicitly says to skip one of them.
