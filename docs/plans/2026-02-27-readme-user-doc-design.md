# README 用户文档设计（2026-02-27）

## 目标
为 GitHub 仓库提供一份面向终端用户的高可用 README，覆盖安装、功能介绍、使用流程、配置说明与常见问题速查。要求支持中英文一键切换，默认中文展示。

## 关键决策
- 受众：终端用户优先。
- 语言策略：`README.md` 作为中文主文；`README.en.md` 提供英文完整文档；两者顶部互链实现“一键切换”。
- 内容深度：采用“详细说明 + 速查 FAQ”模式，避免开发者导向细节干扰初次使用。
- 风格：每个操作写明“怎么做 + 应看到什么 + 失败时如何定位”。

## 信息架构
1. 插件简介
2. 核心能力
3. 安装指南
4. 30 秒快速开始
5. 详细使用说明（入口差异、交互逻辑、分组规则、任务控制）
6. 配置说明（LLM / Prompt / 分组 / 本地规则 / 外观 / 导入导出）
7. 常见问题速查（高频问题 10-15 条）
8. 隐私与数据说明
9. 版本与更新

## 兼容与维护策略
- 文档内容严格对齐当前 UI 文案与行为（如按钮名、分组名、状态提示）。
- 版本锚点跟随当前稳定版本（v1.0.35）。
- FAQ 重点覆盖真实迭代中高频报错与误用场景，减少支持成本。
- 后续每次涉及交互或配置变更时，同步更新 README 对应章节与 FAQ。
